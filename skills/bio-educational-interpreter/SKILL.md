# bio-educational-interpreter

## 适用场景
当用户提供或系统生成生物信息学分析工具（如 BLAST, ClustalW, PyMOL 脚本, GO 富集分析等）的原始文本输出，且需要将其转化为易于理解、具有教育意义的结构化报告时使用。

## 步骤
1. **识别工具与任务**：分析输入数据，确定其来源工具（例如：BLASTp, Multiple Sequence Alignment, 进化树构建）和分析目的。
2. **提取核心指标**：从原始数据中提取关键数值，如 E-value, Identity%, Bit Score, Gap open, p-value 等。
3. **补充背景知识**：简述涉及的生物学对象（基因、蛋白质、物种）的基本功能和在生物学研究中的重要性。
4. **参数深度解读**：解释特定参数在当前上下文中的含义（例如：解释为什么在当前序列长度下 E-value 趋近于 0 代表极强的同源性）。
5. **结果重构与可视化**：使用 Markdown 表格或 Mermaid 流程图重构原始输出，增强可读性。
6. **总结科学意义**：说明该分析结果对后续实验（如引物设计、功能验证）或理论研究的指导意义。

## 工具和命令
- **Markdown**: 用于结构化排版和表格呈现。
- **LaTeX**: 用于书写生物统计公式（如 BLAST 的期望值公式 $E = Kmn e^{-\lambda S}$）。
- **Mermaid**: 用于绘制简单的进化关系或实验流程。
- **生物数据库引用**: 建议在解释中引用 NCBI, UniProt 或 PDB 的标准编号。

## 注意事项
- **术语平民化**：每个专业术语（如 "Ortholog", "Conserved Domain"）在首次出现时应提供简短定义。
- **数值敏感性**：注意区分 "Identity"（一致性）和 "Similarity"（相似性）的细微差别。
- **规范命名**：物种拉丁学名必须使用斜体（如 *Homo sapiens*）。
- **警惕陷阱**：在解读结果时，应提醒用户注意常见的生物信息学陷阱，如“低复杂度区域导致的虚假高分”。

## 示例
**原始输入 (BLAST 结果片段)**:
`Query_1  120  LSD...  180`
`Sbjct_1  450  LSD...  510`
`Identities = 55/60 (91%), Gaps = 0/60 (0%)`
`Expect = 2e-34`

**转化后的教学文档**:
### 1. 分析概览
本次分析使用了 **BLASTp** 算法，将您的查询序列与数据库中的已知蛋白质进行比对。

### 2. 核心指标解读
- **一致性 (Identity): 91%**。在 60 个氨基酸的比对区域中，有 55 个完全相同。这表明两者在进化上高度保守。
- **期望值 (E-value): 2e-34**。这是衡量匹配可靠性的核心指标。该值远小于 $10^{-5}$，意味着这种匹配纯属巧合的概率几乎为零，结果具有极高的统计学显著性。

### 3. 科学意义与建议
如此高的相似性通常意味着这两个蛋白质属于 **直系同源 (Orthologs)**，极可能执行相似的生物学功能。
- **后续建议**：您可以进一步通过 Pfam 数据库查询该序列的保守结构域，以确认其生化活性位点。

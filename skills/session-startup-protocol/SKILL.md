# session-startup-protocol

## 适用场景
当接收到 `/new` 或 `/reset` 指令，或者在任何新会话开始的第一步时使用。用于确保 AI 代理在与用户交互前已完全同步项目上下文、对齐模型状态并进入预设人格。

## 步骤
1. **环境审计**：检查当前运行的模型版本。对比系统提示词（System Prompt）中定义的 `default_model`。如果两者不一致，需记录该差异以便在回复中告知用户。
2. **上下文加载**：在发出第一条回复前，必须主动读取必要的项目文件。通常包括：
   - `SKILLS.md` (加载可用技能)
   - `PROJECT_CONTEXT.md` (了解当前项目进度)
   - 任何在系统指令中明确要求的“启动必读”文件。
3. **人格注入**：根据配置的 Persona（人格）定义，调整回复的语气、措辞、情绪和习惯用语。
4. **生成引导回复**：
   - 长度控制在 1-3 句。
   - 包含符合人格的问候。
   - 如果模型不匹配，提及当前使用的模型。
   - 询问用户接下来的任务目标。
   - **严禁**提及内部处理步骤、读取的文件名、使用的工具或推理过程。

## 工具和命令
- `read_file`: 用于读取 `SKILLS.md` 或其他上下文文件。
- `ls`: 用于扫描当前目录以确认必要文件是否存在。
- `sessions_yield`: (如果适用) 用于同步会话状态。

## 注意事项
- **静默执行**：读取文件和环境检查应在后台逻辑中完成，不要在回复中列出“我正在读取文件...”。
- **简洁性**：首条回复必须极简，避免长篇大论。
- **模型一致性**：如果系统要求使用特定的高级模型（如 Claude 3.5 Sonnet）而当前是其他模型，必须告知用户，以防执行复杂任务时能力不足。
- **禁止泄露**：不要在回复中提及“系统提示词”、“指令集”或“启动序列”等内部术语。

## 示例
**场景**：系统要求默认使用 `claude-3-5-sonnet`，但当前运行在 `gpt-4o`，且已读取完上下文。

**回复内容**：
“你好！我是 SuperClaw。我已经准备好协助你处理项目了（注意：当前运行模型为 GPT-4o，而非默认的 Claude 3.5）。今天我们从哪个模块开始？”

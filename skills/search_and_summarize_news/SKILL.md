# 搜索并总结新闻

## 适用场景
当用户请求获取关于特定主题、事件或一般性新闻的最新信息，并要求进行总结时，可以使用此 Skill。例如，用户可能询问“最近有什么重要新闻？”、“请总结一下关于人工智能的最新进展”或“帮我查找并总结关于气候变化谈判的进展”。

## 步骤
1.  **理解并明确用户需求**:
    *   首先，与用户确认他们感兴趣的新闻主题、时间范围（例如，最近一周、昨天）、地域偏好等。
    *   如果用户请求宽泛，可以主动建议一些热门话题或询问用户是否有特定关注点。
    *   **示例对话**: “您好！您想了解哪个领域的新闻？是关于科技、经济、国际关系，还是其他方面？有没有特定的时间范围，比如最近几天？”

2.  **制定搜索策略并获取用户批准**:
    *   根据用户明确的需求，规划详细的 `web_search` 查询关键词。考虑使用多个查询来覆盖不同方面或细化搜索结果。
    *   在执行搜索前，向用户简要说明搜索计划，例如：“我计划使用关键词‘[关键词1]’和‘[关键词2]’来搜索最近的新闻，您觉得这样可以吗？”
    *   **示例**: “我将搜索‘最新人工智能突破’和‘AI伦理讨论’来获取相关信息。您同意我这样搜索吗？”

3.  **执行 `web_search`**:
    *   使用 `web_search` 工具执行规划好的查询。根据需要，可以进行多次搜索以收集足够的信息。
    *   **注意**: 优先选择信誉良好、权威的新闻来源。

4.  **分析和筛选搜索结果**:
    *   仔细阅读 `web_search` 返回的结果，识别最相关、最权威的文章和信息源。
    *   忽略重复、过时或不相关的内容。
    *   如果初步结果不足或质量不高，返回步骤2或3，调整搜索策略或关键词，并再次执行搜索。

5.  **综合信息并生成总结**:
    *   从筛选出的文章中提取关键事实、观点、数据和事件。
    *   将这些信息组织成一个逻辑清晰、连贯且简洁的总结。
    *   总结应包含主要事件、影响、相关方和未来展望（如果适用）。
    *   确保总结客观、准确，并避免个人偏见。

6.  **向用户呈现总结**:
    *   以自然语言向用户提供最终的总结。
    *   可以询问用户是否需要进一步的细节或澄清。

## 工具和命令

*   **`web_search(query: str, num_results: int = 5, time_range: str = None)`**:
    *   `query`: 必需，用于搜索的关键词或短语。
    *   `num_results`: 可选，指定返回的搜索结果数量，默认为5。
    *   `time_range`: 可选，指定搜索结果的时间范围，例如 "24h", "7d", "30d", "1y"。
    *   **用法示例**:
        ```python
        # 搜索最近24小时内关于“量子计算最新进展”的新闻
        web_search(query="量子计算最新进展", time_range="24h")

        # 搜索关于“全球经济展望”的5个结果
        web_search(query="全球经济展望", num_results=5)
        ```
*   **`sessions_yield(message: str)`**:
    *   在需要用户确认或提供更多信息时，使用此命令暂停执行并向用户发送消息。
    *   **用法示例**:
        ```python
        sessions_yield("我计划搜索关于‘[主题]’的最新新闻，您同意我这样做吗？")
        ```

## 注意事项
-   **关键词精确性**: 模糊或过于宽泛的关键词会导致大量不相关结果。尽量使用具体、精确的短语。
-   **信息过载**: 不要将原始搜索结果直接呈现给用户。务必进行筛选、提炼和总结。
-   **信息偏见与来源**: 警惕新闻来源的潜在偏见。尽量从多个、多样化且信誉良好的来源获取信息，以确保总结的客观性。
-   **时效性**: 对于新闻总结，时效性至关重要。利用 `time_range` 参数确保获取最新信息。
-   **用户确认**: 在执行耗时或可能产生大量结果的搜索前，与用户确认搜索策略，可以有效避免返工。
-   **迭代优化**: 如果第一次搜索结果不理想，不要气馁。根据结果调整关键词、搜索范围或策略，并再次尝试。

## 示例

**用户请求**: “请帮我总结一下最近关于电动汽车行业的重要新闻。”

**执行步骤**:

1.  **理解并明确用户需求**:
    *   Agent: “好的，您想了解最近电动汽车行业的重要新闻。您有没有特别关注的方面，比如技术突破、市场趋势、政策变化，或者特定公司？时间范围是最近几天还是更长？”
    *   用户: “主要关注市场趋势和技术突破，时间范围是最近一周。”

2.  **制定搜索策略并获取用户批准**:
    *   Agent: “好的，我计划使用 `web_search` 搜索关键词‘电动汽车市场趋势 最近一周’和‘电动汽车技术突破 最近一周’。您觉得这样可以吗？”
    *   用户: “可以。”

3.  **执行 `web_search`**:
    ```python
    # 搜索市场趋势
    web_search(query="电动汽车市场趋势 最近一周", time_range="7d", num_results=5)
    # 假设返回结果包含关于销量增长放缓、价格战等信息

    # 搜索技术突破
    web_search(query="电动汽车技术突破 最近一周", time_range="7d", num_results=5)
    # 假设返回结果包含关于新型电池技术、充电速度提升等信息
    ```

4.  **分析和筛选搜索结果**:
    *   Agent 阅读并筛选 `web_search` 返回的多个新闻链接和摘要，识别出关于特斯拉降价、中国市场竞争加剧、固态电池研发进展、快充技术应用等核心信息。

5.  **综合信息并生成总结**:
    *   Agent 将提取的信息进行整合和归纳。

6.  **向用户呈现总结**:
    *   Agent: “根据我最近一周的搜索，电动汽车行业有几个重要趋势和技术突破：
        *   **市场趋势方面**：全球电动汽车销量增速有所放缓，尤其是在部分成熟市场。价格竞争日益激烈，一些主要厂商如特斯拉在中国市场采取了降价策略以刺激销量。同时，供应链稳定性依然是行业关注的焦点。
        *   **技术突破方面**：固态电池技术研发持续取得进展，一些公司宣布了新的里程碑，预示着未来电动汽车续航里程和安全性的提升。此外，超快充电技术也在不断普及，显著缩短了充电时间。
        *   **政策与基础设施**：多国政府继续推出支持电动汽车发展的政策，充电基础设施建设也在加速推进。
        您对这个总结满意吗？或者想了解更多细节？”

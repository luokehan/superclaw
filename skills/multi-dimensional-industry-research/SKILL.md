# 多维度行业深度调研

## 适用场景
- 需要对新兴技术领域（如 AIGC、人形机器人、低空经济等）进行快速、全面的行业摸底。
- 需要为投资决策、产品立项或政策研究提供结构化的数据支持。
- 当用户提出的调研需求较为笼统，需要自动拆解为可执行的搜索子任务时。

## 步骤
1. **定义调研主体**：明确技术领域或行业关键词，并确定调研的时间跨度（通常为近 1-3 年）。
2. **多维度拆解**：将调研任务自动拆分为以下五个核心维度：
    - **市场规模**：查找最新市场估值、复合年增长率（CAGR）及主要增长驱动力。
    - **融资/资本**：搜集近期大额融资事件、活跃投资机构及估值独角兽。
    - **学术/技术热度**：检索顶级会议（如 CVPR, NeurIPS）趋势、arXiv 热门论文及核心专利分布。
    - **商业化进度**：识别标杆产品、典型应用场景（Use Cases）及头部企业的落地现状。
    - **就业/人才影响**：分析相关岗位的招聘需求变化、薪资水平及所需核心技能。
3. **执行并发搜索**：利用 `web_search` 工具，针对上述每个维度生成 2-3 个精准搜索指令。
4. **数据清洗与交叉验证**：对比不同来源的数据（如 Gartner vs IDC），标注数据冲突点，提取具体数值。
5. **构建对比矩阵**：将收集到的碎片化信息填入结构化表格，进行横向或纵向对比。
6. **撰写深度报告**：按照“核心结论 - 维度详解 - 对比表格 - 趋势预测”的结构输出。

## 工具和命令
- **搜索指令模板**：
  - `web_search(query="[领域名称] 2024 市场规模 增长率 报告")`
  - `web_search(query="[领域名称] 2024 融资事件 投资机构 汇总")`
  - `web_search(query="[领域名称] 核心技术 瓶颈 商业化挑战")`
- **数据处理**：使用 Markdown 表格语法整理数据。

## 注意事项
- **时效性陷阱**：前沿技术变化极快，必须在搜索词中加入当前年份（如 2024），并优先采用近 6 个月内的信息。
- **数据孤岛**：不要只看中文资料，必须包含英文搜索以获取全球学术和资本动态。
- **避免泛泛而谈**：报告中必须包含具体的公司名称、具体的融资金额、具体的论文题目或具体的技术参数。
- **信源权重**：官方统计局 > 知名咨询机构（麦肯锡、Gartner 等） > 行业垂直媒体 > 个人博客。

## 示例
**任务**：调研“AI Agent（人工智能代理）”行业。

**执行过程**：
1. 拆解为：AI Agent 市场规模预测、2024 AI Agent 融资名单、AutoGPT/BabyAGI 学术引用情况、企业级 AI Agent 落地案例、AI Agent 工程师招聘趋势。
2. 搜索并提取数据。
3. 生成对比表：

| 维度 | 关键指标/现状 | 核心代表/数据 |
| :--- | :--- | :--- |
| **市场规模** | 预计 2030 年达到 $XX B | CAGR 预计超过 30% |
| **资本热度** | 早期融资密集 | OpenAI, Anthropic, Adept |
| **技术瓶颈** | 幻觉问题、长短期记忆 | 核心框架：LangChain, AutoGPT |
| **商业化** | 从个人助手向企业工作流转型 | 典型场景：自动化客服、代码生成 |

4. 输出包含趋势预测的完整报告。
